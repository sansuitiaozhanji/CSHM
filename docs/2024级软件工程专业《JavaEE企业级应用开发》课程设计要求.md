# 2024级软件工程专业《JavaEE企业级应用开发》课程设计要求

---

## 一、题目

**校园二手物品交易系统的设计与实现**

---

## 二、功能要求

系统主要用户是游客、注册用户（学生/教职工注册用户，即买卖双方）、平台管理员。

### （一）游客的主要功能

1. 浏览系统首页（首页可自行设计，例如显示轮播图、热门二手商品）
2. 查看交易须知、平台规则、帮助中心、公告资讯等；
3. 注册账号；
4. 登录系统；

### （二）注册用户的主要功能

#### 1. 基础账号功能

1. 登录系统、退出系统，登录和退出系统时有日志记录
2. 修改个人资料

#### 2. 买家功能需求

1. 搜索商品、收藏商品、加入购物车、提交订单、确认交易
2. 在线私信卖家沟通价格、交易时间地点
3. 订单管理：待付款、待自提、已完成、已取消订单
4. 交易完成后评价商品、查看历史购买记录

#### 3. 卖家功能需求

1. 发布二手闲置商品、编辑商品信息、下架商品、重新上架、删除商品
2. 管理售卖订单，确认交易、约定校园面交
3. 查看买家留言与咨询，回复私信
4. 查看个人在售商品、历史售出记录
5. 设置商品自提地点、交易时间

### （三）交易说明

交易采用模拟方法，无需接入支付宝或微信交易。要求给每个注册用户创建一个平台虚拟余额，所有交易都在平台内部用余额完成，不涉及真实货币。

#### 1. 账户初始化

1. 新用户注册成功后，系统自动给虚拟账户发放初始余额（例如：1000元虚拟币）。
2. 余额仅在平台内使用，无法提现、不能兑换真实货币。

#### 2. 支付模拟流程（买家视角）

1. 买家提交订单后，系统显示"支付"按钮。
2. 点击支付 → 校验买家虚拟账户余额是否 ≥ 订单金额：
   - **余额充足**：直接扣减买家账户余额，订单状态从「待付款」变为「待自提」，同时卖家账户余额增加对应金额（注意交易完成后物品款项才会正式入账）。
   - **余额不足**：提示"虚拟余额不足，请联系管理员充值或取消订单"。
3. 交易完成（买家确认收货）后，卖家账户余额正式入账，订单变为「已完成」。

### （四）平台管理员功能需求

#### 1. 用户管理

1. 查看注册用户信息
2. 禁用违规账号

#### 2. 商品管理

1. 商品分类管理
2. 审核用户发布所有二手商品，审核通过/驳回
3. 强制下架违规物品（违禁品、危险品等）

#### 3. 订单管理

1. 查看平台所有交易订单，监控交易流程

#### 4. 内容管理

1. 发布系统公告、平台规则、校园交易资讯等
2. 管理首页轮播图、热门商品推荐设置
3. 管理用户评论、删除违规评论、屏蔽不良言论

#### 5. 日志管理

1. 浏览日志、搜索日志、导出日志

#### 6. 数据统计管理

1. 统计校园二手物品数量、不同状态的订单数量、成交率等
2. 用户数据统计：总注册人数、正常账号、禁用账号数量统计

> 功能不限于以上所列出的，若有扩展功能，可加分。

---

## 三、技术要求

1. 请根据实际业务需求自行设计数据库，数据库采用 SQL Server 或 MySQL。
2. 后端开发利用面向对象程序设计方法（**必须**），采用 MVC 模式（**必须**），利用 JSP+JavaBean+Servlet 技术实现或者（Spring+SpringMVC+MyBatis）或者（SpringBoot + MyBatis）等框架实现。
3. 前端技术不限，可以使用 HTML+CSS+JavaScript，也可以使用 JQuery、Vue.js 等 JavaScript 框架，具有友好的 UI 界面，系统操作简单、易用。
